# 数据 Schema 规则

本文件定义 financial-researching skill 内部的统一数据表结构规范。
所有脚本的输入输出数据必须符合本规范。

---

## 一、通用列命名规则

| 语义 | 标准列名 | dtype | 说明 |
|------|---------|-------|------|
| 交易日期 | `date` | `datetime64[ns]` | 格式 YYYY-MM-DD |
| 证券代码 | `code` | `str` | 6 位纯数字，如 `"000001"` |
| 开盘价 | `open` | `float64` | 单位：元 |
| 最高价 | `high` | `float64` | 单位：元 |
| 最低价 | `low` | `float64` | 单位：元 |
| 收盘价 | `close` | `float64` | 单位：元 |
| 成交量 | `volume` | `int64` | 单位：股（非手） |
| 成交额 | `amount` | `float64` | 单位：元 |
| 日收益率 | `return` | `float64` | 小数形式，0.05 = 5% |
| 市盈率 | `pe` | `float64` | TTM 口径 |
| 市净率 | `pb` | `float64` | — |
| 市值 | `market_cap` | `float64` | 单位：元 |
| 行业 | `industry` | `str` | 申万一级行业 |

---

## 二、缺失值处理规则

| 列类型 | 处理方式 | 理由 |
|--------|---------|------|
| 价格列（open/high/low/close） | `ffill()`（前值填充） | 停牌期间价格不变 |
| 成交量（volume） | 填 `0` | 停牌无成交 |
| 成交额（amount） | 填 `0` | 同上 |
| 收益率（return） | 填 `0.0` | 停牌收益为零 |
| 基本面指标（pe/pb/market_cap） | 保留 `NaN` | 不可随意插值 |
| 行业（industry） | 保留 `NaN` | 无法推断 |

**建模前必须执行 `dropna()`**，确保传入 linearmodels 的数据无 NaN。

---

## 三、面板数据表结构（供建模使用）

面板数据用于 linearmodels 回归，必须满足以下要求：

### 索引

- **MultiIndex**：`(code, date)`
- `code` 在外层（Level 0），`date` 在内层（Level 1）

### 必要列

```
code | date       | return | pe   | pb   | market_cap | ...自定义因子
-----|------------|--------|------|------|------------|-------------
000001 | 2023-01-03 | 0.012 | 15.3 | 1.2  | 3.5e11     | ...
000001 | 2023-01-04 | -0.003| 15.1 | 1.19 | 3.48e11    | ...
000002 | 2023-01-03 | 0.008 | 22.1 | 2.3  | 1.2e11     | ...
```

### 要求

1. 同一 `(code, date)` 组合不能有重复行
2. 面板必须是 **非平衡面板**（允许不同股票有不同的时间范围）
3. 所有回归变量必须是数值型
4. 分类变量（如行业）需提前转为 dummy 变量

---

## 四、行情数据表结构（供回测使用）

backtrader 数据馈送要求的表结构：

### 必要列

```
date       | open  | high  | low   | close | volume
-----------|-------|-------|-------|-------|--------
2023-01-03 | 13.50 | 13.85 | 13.40 | 13.72 | 5832100
2023-01-04 | 13.75 | 14.10 | 13.60 | 13.95 | 6215300
```

### 要求

1. **必须使用复权价格**（推荐前复权）
2. `date` 列必须为 `datetime` 类型
3. 数据按 `date` 升序排列
4. 不允许有 NaN（backtrader 无法处理）
5. 如有额外列（如信号列），backtrader 可通过自定义 Data Feed 读取

---

## 五、面板表与行情表的关键差异

| 维度 | 面板表（建模用） | 行情表（回测用） |
|------|-----------------|-----------------|
| 索引 | MultiIndex (code, date) | date 为索引或普通列 |
| 股票数 | 多只 | 通常单只（每只单独馈送） |
| 价格复权 | 不强制 | 强制前复权 |
| NaN 处理 | dropna 后传入 | 完全不允许 NaN |
| 额外列 | 因子、控制变量 | 可选信号列 |
| 排序 | 按 (code, date) | 按 date 升序 |
